# v0.4.0 — Runtime modernization release

## Summary

- repository-level release;
- closes Roadmap 3 runtime modernization;
- genre-classifier production runtime modernized;
- legacy MusiCNN production contract preserved.

## What changed

- production runtime moved from Python 3.6 / TensorFlow 1.15 to Python 3.12 + TensorFlow 2.21 + essentia-tensorflow;
- genre-classifier Docker runtime was updated in Roadmap 3.11;
- post-switch stabilization completed in Roadmap 3.12;
- release notes are repository-level documentation for that milestone.

## Runtime after v0.4.0

- Python 3.12.13;
- TensorFlow 2.21.0;
- essentia-tensorflow 2.1b6.dev1389;
- Essentia 2.1-beta6-dev;
- numpy 2.4.4;
- protobuf 7.34.1;
- h5py 3.14.0;
- FastAPI 0.83.0;
- Pydantic 1.10.26;
- Uvicorn 0.16.0.

## Production contract

- default provider remains legacy_musicnn;
- /classify contract unchanged;
- response shape unchanged:
  - ok
  - message
  - genres
  - genres_pretty
- runtime shadow disabled by default;
- tidal-parser unchanged.

## What did not change

- no provider switch;
- no canary rollout;
- no LLM production adoption;
- no LLM cutover;
- no migration to lighter non-TensorFlow model;
- no /classify contract changes;
- no response shape changes;
- no tidal-parser behavior changes.

## Validation

## Import-order invariant

- supported production path: app-first / Essentia-first;
- TensorFlow-first mixed import path remains unsupported and documented;
- duplicate Bitcast / RegisterAlreadyLocked did not appear in normal startup/classify path.

## Compatibility

- existing /classify consumers do not need changes;
- tidal-parser does not need changes;
- response parsers do not need changes.
