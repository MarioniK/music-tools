# v0.3.0 — genre-classifier LLM migration foundation

## Summary

v0.3.0 фиксирует завершение Roadmap 2 LLM migration foundation для `genre-classifier`.

Это migration milestone release. Это не LLM production cutover, не canary rollout и не default-provider switch. Production behavior остаётся legacy-safe: default provider remains `legacy_musicnn`, production response remains legacy-only, external `/classify` contract unchanged, response shape unchanged, `tidal-parser` unchanged.

Canary rollout, production cutover и Python/runtime upgrade остаются out of scope для v0.3.0.

## What changed

В рамках Roadmap 2 для `genre-classifier` были подготовлены и зафиксированы:

- provider abstraction;
- schema validation and compatibility mapping;
- LLM provider scaffold;
- local LLM runtime integration baseline;
- prompt discipline and controlled vocabulary;
- offline comparative evaluation;
- runtime shadow-readiness;
- runtime shadow execution baseline;
- controlled live shadow evidence;
- release-readiness decision artifact.

Эти изменения формируют foundation для дальнейшей LLM migration work, но не переводят production traffic на LLM.

## What did not change

- default provider remains `legacy_musicnn`;
- `/classify` contract unchanged;
- response shape unchanged;
- production responses remain legacy-only;
- `tidal-parser` unchanged.

## Known gaps

Known gaps after v0.3.0:

- `invalid_output` live evidence gap remains documented;
- no actual canary;
- no default-provider switch approval;
- Python/runtime technical debt remains post-v0.3.

These gaps do not block v0.3.0 as a migration milestone release with legacy default behavior. They do block or constrain future production LLM adoption, including any canary rollout, provider switch, or cutover approval.

## Out of scope

Explicitly out of scope for v0.3.0:

- canary rollout;
- production cutover;
- default-provider switch;
- LLM as source of truth;
- API changes;
- response shape changes;
- `tidal-parser` changes;
- Docker/Python runtime upgrade;
- dependency refresh.

## Operational notes

- default deployment remains legacy-safe;
- shadow execution must be enabled explicitly;
- shadow execution must not affect production response;
- release was validated with build, health check, logs review, and baseline `/classify` smoke.

## Rollback notes

- runtime rollback should normally not be required because default remains `legacy_musicnn`;
- if shadow was enabled manually, disable shadow-related config and restart `genre-classifier`;
- docs-only commits can be reverted normally.
