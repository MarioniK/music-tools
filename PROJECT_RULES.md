# Project Rules

## Общая логика
- `tidal-parser` — главный сервис проекта.
- `genre-classifier` — вспомогательный сервис анализа аудио.
- Финальные решения о жанрах и блоговом выводе принимает `tidal-parser`.

## Источники данных
- Discogs → жанры релиза
- MusicBrainz → год, страна, тип релиза
- Genre Classifier → аудио-жанры

## Правила разработки
- Не хранить `.env` в git
- Не хранить `cache.db` в git
- Не хранить временные файлы в git
- Не редактировать код напрямую внутри контейнеров
- Все изменения делать в файлах проекта и применять через `docker compose`

## Кэш
- Кэш не должен ухудшать хороший найденный результат
- Новый слабый матч не должен затирать старый сильный матч
- Кнопка сброса кэша должна быть безопасной

## Жанры
- `genre-classifier` отдаёт сырой и мягко нормализованный набор жанров
- Финальная склейка жанров живёт в `tidal-parser`
- Общие и дублирующиеся жанры желательно чистить на финальном слое

## UI
- В `tidal-parser` не показывать пустые или дублирующие поля
- Поле `entity_type` = тип ссылки (`track` / `album`)
- Поле `release_kind` = тип релиза (`single` / `album` / `ep`)
